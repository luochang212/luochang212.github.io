// 01 问题背景 background

在欧洲旅行时，如何走最少的路遍历所有景点？Google Maps 的 "Add destination" 只能按顺序串联地点，无法给出全局最短路径。

这个需求正是运筹学经典问题——旅行商问题（TSP）：给定一组城市，找出一条遍历所有城市并回到起点的最短回路。

  • >输入
    一组 POI（景点）名称 + 所在城市
  • >输出
    Google Maps 上的最短遍历路线 + 总距离

// 02 技术方案 stack

  • >坐标获取
    googlemaps 包调用 Google Maps API，获取每个 POI 的经纬度
  • >TSP 求解
    Google OR-Tools 求解器，构建距离矩阵并计算最短哈密顿回路
  • >路线可视化
    gmaps 包在 Jupyter Notebook 中渲染交互式地图，标注路线
  • >运行环境
    Jupyter Notebook + Python，支持 walking / driving / bicycling / transit 四种出行方式

// 03 使用指南 guide

克隆项目后在 Jupyter Notebook 打开 TSPSolver.ipynb，修改第二个代码块的配置变量即可：

  • >places
    目的地列表，如 'London Eye', 'Big Ben', ...
  • >Location
    所在城市，如 'London'
  • >Mode
    出行方式："walking" / "driving" / "bicycling" / "transit"
  • >password
    Google Maps API key，从 Google Cloud Console 获取

从头到尾运行所有代码块，即可在 Notebook 中获得交互式最优路线图。
