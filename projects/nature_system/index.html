// 01 建模方法 approach

Agent-Based Model 以个体为中心建模——虚拟实体与现实个体一一对应，属性是对现实特征的抽象，行为按既定规则迭代演化。与 Equation-Based Model 关注系统平均状态不同，ABM 擅长表达每个个体身上发生了什么，尤其适合以研究个体行为规则为目的的建模。

本模型包含三种 Agent，各有不同属性与角色分工：

  • >狼（Wolf）
    分头狼与普通狼。头狼拥有集群行为决策权——根据上一轮捕猎结果判断迁徙方向（吃到 → 朝成功狼移动；没吃到 → 随机探索）。头狼还维护 population（群规模）、range（活动范围）、eaten（捕食数）等群级属性。普通狼在头狼活动范围内随机分布，不参与决策
  • >麋鹿（Moose）
    以草为食，位置草被吃尽后迁徙。被捕食概率与狼的距离成反比：p = 1 - d/spd
  • >草（Grass）
    均匀平铺于地图，被吃后不生长，驱动麋鹿持续迁徙寻找新食物源。简单但关键的底层驱动力

// 02 行为规则 behaviors

每种 Agent 每轮迭代按 eat → migrate → die → breed 顺序执行四类行为：

  • >捕食（eat）
    狼：麋鹿出现在搜索范围内即判定，距离越近捕获概率越高（pk = 1 - d/spd，随机数判定）。麋鹿：吃所在位置的草，food 属性 ± 1
  • >迁徙（migrate）
    头狼利用局部信息决策（吃到麋鹿 → 朝成功狼移动；没吃到 → 随机探索）。大狼群更活跃，小狼群更谨慎。麋鹿仅当所在地没草时才移动
  • >死亡（die）
    狼：饿死或老死（max 25 轮）；狼群共享 food，为 0 时死亡一只普通狼。麋鹿：饿死、老死或被吃。头狼死后由普通狼继任，保证狼群延续
  • >生育（breed）
    麋鹿每次生 1 只（间隔 ≥ 10 轮，food > 10）。狼群以群为单位生育，数量与群规模成正比、与食物富余程度挂钩。生育后母体食物分给孩子

// 03 技术架构 architecture

采用 MATLAB 面向对象编程，总计 34 个 m 文件、1846 行代码。 主函数 ecolab.m 搭建基本参数与数据结构后，通过核心调度器 agent_solve 按序调用各 Agent 的行为函数。 全局变量 MESSAGES 维护迭代间状态传递——只要 MESSAGES 和 agent 在每轮正常更新，模型就能稳定运行。

调用命令：ecolab(30, 100, 5, 4, 60, false, false) — 环境 30×30，100 只麋鹿，每群 5 只狼，4 个狼群，迭代 60 轮。
