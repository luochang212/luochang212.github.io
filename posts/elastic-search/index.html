Elasticsearch 是一个开源的分布式全文搜索引擎。它建立在当前最先进有效的搜索引擎库 Lucene 之上。Elasticsearch 集成了 Lucene 的功能，并通过一套简单的 API 隐藏了 Lucene 的复杂性，是一个开箱即用的搜索引擎。

初识 Elasticsearch

不仅仅是搜索

ES 不仅仅能够搜索，它还提供了一套分布式实时存储文档的解决方案。它允许用户将数据存储在多个服务器节点上。这些节点能实时共享数据。一旦主节点的内容发生改变，它发生的改变会实时传递到其它副节点上。如果主节点挂掉，它还能瞬间选举出一个副节点来充当主节点，保障了软件的可靠性。

分布式存储不仅能保障数据安全，而且可以加快检索速度。利用 MapReduce 模型，ES 可以在多个节点上并行地检索数据。

领域专用语言

前文已经提到，ES 为全文检索提供了一套简单的 API。为了统一查询语句，ES 设计了一套专门用于查询的领域专用语言(DSL, domainspecific language)。这套语言基于 JSON，语法比较简单，容易上手。但如果遇到复杂的多条件的查询，查询体会写得非常冗长，这也可以说是 ES 的一个小坑。

SQL 和 NoSQL

SQL 全称 Structured Query Language。SQL 中的 S 突出了它是结构化的查询语言。结构化意味着数据是受字段限制的。比如现在有一组结构化的数据，已知它有3条数据、5个字段，那么我们一定可以确定它是一张3乘5的数据表。但 NoSQL 就并非如此。NoSQL 允许更灵活多变的数据结构。数据可以有不同的字段以及复杂的层级关系。NoSQL 的数据可以用 JSON 来表达。

入门 Elasticsearch

安装和运行

  1. 首先打开https://www.elastic.co/downloads/elasticsearch，下载操作系统对应版本的 ES。

  2. 解压下载的文件，然后将解压出的文件夹放在你希望放置的目录下。

  3. 打开 Terminal (MacOS) 或者 PowerShell (Windows)，在命令行中切换到解压出的文件夹所在的目录。运行以下命令：

    cd elasticsearch-7.4.0
cd bin
./elasticsearch

    这将在当前 Terminal 窗口运行 ES，如果你不希望 ES 服务关闭，请勿关闭窗口或者中止程序。

  4. 打开一个新的 Terminal 窗口，然后运行：

    curl 'http://localhost:9200/?pretty'

    如果 ES 运行正常，终端将收到一条包含本地集群信息的 JSON 数据。

Note: 如果希望获取更多关于安装的介绍，可以参考 安装并运行 Elasticsearch 或 How to install Elasticsearch on MacOS。

Elasticsearch 文档

文档在 ES 中，是一个特定的术语。它代表用于存储对象的存储单元。一个对象通常来说不会是简单的键值对，它可能包含复杂的数据结构，比如日期、地理数据、数组等。为表达这种复杂的，甚至多层次的数据结构，ES 使用 JSON 作为文档存储的固定格式。因此在 ES 中，一个 JSON 文档就代表一个存储对象。

一个文档除了包含它的数据，ES 规定它还必须包含三个元数据：

  • _index：文档在哪存放
  • _type：文档表示的对象类别
  • _id：文档的唯一标识

文档中的其他数据，可以视需求定义。
