从泊松函数讲起

泊松分布 表示在给定时间段内发生给定数量的事件的概率。这个定义比较抽象。举个具体的例子，假设你每小时接到电话的概率是固定的，比如每小时 0.05 个，那么你在接下来 1 小时内接到电话个数的概率，就服从泊松分布:

1 小时内接到 0 个电话，对应一个概率值 $P_0$；

1 小时内接到 1 个电话，对应一个概率值 $P_1$；

… …

1 小时内接到 n 个电话，也对应一个概率值$P_n$。

这些概率值组成一个概率分布列，它们的值 $(n, P_n)$ 在二维坐标下连成一条曲线。这条曲线所在的函数就是泊松分布的概率密度函数。其公式及图像如下：

$$\boxed{P(k | t, \lambda)=\frac{(\lambda t)^{k}}{k !} \exp (-\lambda t)}$$

从公式中，我们可以看出：只要确定了 $\lambda$ 和 $t$，该式就退化成了概率$P$ 关于事件发生次数 $k$ 的函数。 因此，如果我们对不同 $k$ 对 $P$ 值的影响感兴趣，那么确定 $\lambda$ 和 $t$ 的值就至关重要。

$\lambda$, $k$, $t$ 的定义：

  • $\lambda$: 单位时间内，事件发生的频率。
  • $k$: 事件发生次数。
  • $t$: 观测事件发生次数的时间范围。

$\lambda$, $k$, $t$ 三者的关系：

泊松分布衡量的是多长时间内，某事件发生多少次的概率。这里的多长时间指代的是 $t$；事件发生多少次指代的是 $k$。$\lambda$ 则类似该事件的一个固有属性，$\lambda$ 越大，可以简单理解为该事件在一段时间内发生的概率越大。

Note: $t$ 的量纲必须和 $\lambda$的量纲相对应。如果 $\lambda$ 的单位是 次/小时，则 $t$ 也应该取 小时 为单位。

引入一个实例

接下来我们用一个实例，解释泊松分布是如何运用的。

一个淘宝客服，平均每 10 分钟接听 1 个电话，那么他连续 1 小时没接到电话的概率是多少？

$$P\left(k=0 | t=1, \lambda=6\right)=\exp \left(-6\right)=0.0024$$

其中，$t = 1 小时，\lambda = 6 次/小时$。我们发现，这个概率是极低的。这说明这位客服也许正在加夜班，或者他的电话坏了。否则正常情况下，该事件发生的概率只有千分之二，是极其稀罕的。

泊松分布的本质

尽管泊松分布的形式看起来很复杂，但它本质上其实非常简单。泊松函数的隐含假设可以追溯到一个简单的公式：

$$\boxed{P = \lambda \Delta t}$$

这个公式看起来太过简单，以至于你可能不相信它能推导出上文中那个复杂的泊松分布函数。如果你想了解推导过程，可以看我之前写的一篇博客，里面有详细证明 排队论在网络性能分析中的应用。本文的主题不在此，就不展开讲了。

泊松分布的仿真

这个简单公式，就已经包含了泊松分布的核心假设。也就是说，我们只需要用到这个简单公式，就可以对泊松分布做仿真了。

但是这里有一个技巧。因为在假设中，$\Delta t$ 代表无限小的时间间隔，但实际上我们没法用代码编写出无限小这种东西。因此只能量力而行。比如说，如果 $t$ 代表一个小时，而 $\lambda$ 表示 20 次每小时，那么我们的时间间隔只要切到比 20 这个数字大一到两个数量级即可。20 是十这个数量级的，这里我们把一小时切分为一千个小时间片，并将每个时间片内事件发生的概率设为 20⁄1000 ，就满足能$P = \lambda \Delta t$这个公式了。时间片当然是分得越细越好，这里的一到两个数量级，也只是个人之见，具体效果可以自行试验。

OK，闲言少叙，我们这就开始仿真建模。

# -*- coding: utf-8 -*-
"""泊松仿真
   author: github@luochang212
   usage: python poisson.py [RATE] {TIME}
"""

import sys
import random
import collections
import matplotlib.pyplot as plt


class Poisson:
    def __init__(self, rate=sys.argv[1]):
        """init"""
        self.rate = int(rate)

        self.time = 1  # 单位时间
        if len(sys.argv) > 2 and sys.argv[2] != "":
            self.time = int(sys.argv[2])  # 手动指定时间范围

        self.EXP_NUM = 100000  # 实验次数
        self.NUM_LEVEL = 2  # 数量级

    def generator(self, prob):
        """仿真结果生成器"""
        while True:
            if random.random() < prob:
                yield 1
            else:
                yield 0

    def conduct_one_experiment(self, rate, time):
        """进行一次实验
           每次实验中，时间分片的数量比rate高两个数量级
           return: 本次实验中，事件发生的次数
        """
        level = len(str(rate))
        shard_num = 10 ** (level + self.NUM_LEVEL)  # 计算时间分片的数量

        gen = self.generator(rate / shard_num)

        cnt = 0
        for _ in range(time * shard_num):
            cnt += next(gen)
        return cnt

    def get_distribution(self, exp_num, rate, time):
        """多次实验，得到分布
           exp_num: 实验次数
        """
        lst = []
        for _ in range(exp_num):
            lst.append(self.conduct_one_experiment(rate, time))

        return sorted(collections.Counter(lst).items(), key=lambda e: e[0])

    def draw(self, sorted_list):
        """画图"""
        s = sum([e[1] for e in sorted_list])
        x = [e[0] for e in sorted_list]
        y = [e[1] / s for e in sorted_list]

        plt.plot(x, y)
        
        plt.xlabel("k")
        plt.ylabel("P(k)")
        plt.show()

    def main(self):
        """main"""
        sorted_list = self.get_distribution(self.EXP_NUM, self.rate, self.time)
        self.draw(sorted_list)

    @staticmethod
    def calculator(rate, t, k):
        """用于计算泊松函数的概率 P(k|t,lambda)
           rate: lambda
           t: t
           k: k
        """
        import math
        return (rate * t) ** k / math.factorial(k) * math.exp(-rate * t)


if __name__ == '__main__':
    p = Poisson()
    p.main()

泊松分布在生活中的应用

停车问题

公司后门有片停车场，到晚了是这么一副光景👇 想把车停进去可太难了。

全是车！

如果遇到一个好车位，停进去也就罢了。但有时候车位挺多的，但因为害怕走完全程都遇不到一个好车位，还要回头！所以随便找个车位就把车停了。巨亏！

这时候泊松分布就能帮大忙了。我们可以制定这样一种策略：我们把这条路平均分为 2 段。前半段用来观测，后半段用来停车。

假如我们走完前半段路，发现有 5 个车位看起来勉强能把车停进去。那么我们就可以非常粗略地估计，后半段路中车位的数量服从 $\lambda = 5$ 的泊松分布。通过查阅上面图像，我们发现 $k = 0$ 即后半段路中完全遇不到类似车位的概率，只有千分之七左右。

这是非常低的概率，那么我们就有信心在后半段路中找到一个好车位了。这时我们对车位的标准就可以相应提高，非好车位不停！
