未来的钱，可能没你想的那么值钱，这是一件挺反直觉的事。要解释这一切，还要从折现率说起。

GitHub项目地址：insurance.ipynb

有位朋友买保险，顾问给的方案是缴纳 30 年，身故后返还数值多于缴纳额的现金，期间还有保险功能。听起来是纯捡便宜，这究竟是怎么一回事呢？

这是销售给的方案：

  • 费用：分 30 年支付，每年支付 9000 元，共计 27 万元
  • 权益：终生享受重疾险；身故后保险公司一次性返还 50 万元

乍看之下很划算，但是不是真划算，要算过才知道。

很抱歉，为了把问题讲明白，我还得先引一下背景知识。

一、折现率和净现值

销售给的方案，可以用这个表格表示：

0 1 2 3 … 29 30+（以身故时间为准）
-9,000 -9,000 -9,000 -9,000 … -9,000 +500,000

可以看到，每笔钱入账时间不同。为衡量资金的时间成本，我们需要 折现率 和 净现值 两个概念。

1.1 什么是折现率

折现率 (Discount rate) 是把未来的钱折成现在的钱的折价比率。举个简单的例子，如果你认为一年后的 107 元与现在 100 元相当，那么折现率就是 7%，用公式表达为：

$一年后107元的现值 = \frac{107}{1 + 折现率} = \frac{107}{1 + 7\%} = 100$

1.2 什么是净现值

净现值 (Net present value)，简记为 NPV。指未来现金流入现值与未来现金流出现值的差额，说人话就是收入的现值减去支出的现值。

现值随折现率确定而确定，所以我们需要先约定一个折现率，才能让计算继续下去。我们假设折现率为 0.02，则有如下 NPV 计算公式：

$NPV = -9000 -9000*\frac{1}{1 + 0.02} -9000*\frac{1}{(1 + 0.02)^{2}} + ... + -9000*\frac{1}{(1 + 0.02)^{29}} + 500000*\frac{1}{(1 + 0.02)^{t}}$

其中 t 为从交保到身故的间隔年数，值域为 $[30, +\infty]$

根据 NPV 公式，有如下 Python 函数，后面会多次用到它。

# 计算成本、收益、NPV
def calc_npv(discount_rate=0.02, t=30):
    """返回 成本的现值、收益的现值、净现值NPV
    """
    cost = sum([9000/((1+discount_rate)**i) for i in range(30)])
    profit = 500000/((1+discount_rate)**t)
    return cost, profit, profit - cost

二、成本-收益曲线

因为 NPV = 收益的现值 - 成本的现值，所以展示 收益的现值 和 成本的现值 会更直观

下图是用偏保守的 2% 折现率计算的。

横坐标是从买保险到身故的时间间隔（单位年），纵坐标是成本和收益的现值（单位元）。横坐标的起点是 30 年，即以完成 30 期缴费的时刻作为起点。

cost 是一条直线其实不难理解，因为 cost 的计算不含参数 t. 由于 t 是横坐标，这就是说 cost 不随 t 变化而变化。

图中收支平衡点约在 45 年处，假设 30 岁签约保险，那么此时 75 岁还未及平均寿命。别忘了这是用 2% 保守折现率计算的结果，如果用更接近真实的折现率，“收支平衡”的年限还会提前。

画图代码：

# 成本/收益曲线
import matplotlib.pyplot as plt

discount_rate = 0.02
npv = [calc_npv(discount_rate, t) for t in range(30, 60)]
x_list = list(range(30, 60))
cost_list = [e[0] for e in npv]
profit_list = [e[1] for e in npv]

plt.plot(x_list, cost_list, label='cost')
plt.plot(x_list, profit_list, label='profit', marker='.')
plt.title('cost-profit curves')
plt.xlabel('(years)')
plt.ylabel('(yuan)')
plt.legend()
plt.show()

Note: 为什么说 2% 的折现率保守？

时处 2023 年，这是一些证据：

  • 中国青年报：过去十年CPI涨幅是 2%
  • 货币基金年化收益率约 2%
  • 四大行 3 年期定存约 2.6% - 2.8%
  • 地方银行 5 年期定存可达 4%（但有下降趋势）
  • 美元存款定存可达 5% - 7%（但有下降趋势）

三、不同折现率的NPV曲线

2% 的折现率设得太低，我们来看看 3%, 5%, 7% 的折现率下 NPV 的表现。

注：为便于表达，假设 30 岁时签约保险

看看区别：

  • 从收支平衡的时间看：
    • 3% 折现率，65 岁收支平衡
    • 7% 折现率，在缴满之前 NPV 就为负，收支平衡点在缴满前
  • 假设 80 岁身故，那么：
    • 3% 折现率，亏 6.7 万元
    • 7% 折现率，亏 10.2 万元

但是上面说的“收支平衡”也好，“亏”也好，都没有计算保险成本。实际在你亏的数字上，扣除保险公司支付给你的保险费用，才是保险公司赚的利润。但话说回来，50万返款肯定不是无条件给的，对身故的情形有限制，这又多了很多可以扯皮的地方。

那我又有一个脑洞了，万一遗产税出台呢？这 50 万相当于帮后代合理避税了。有条件的老哥不如赌一把？看能不能钻尚未出台的遗产税的空子。

画图代码：

# 不同折现率，对应的净现值曲线
x_list = list(range(30, 60))
get_npv_list = lambda e: [calc_npv(e, t)[2] for t in range(30, 60)]
dr_02 = get_npv_list(0.02)
dr_03 = get_npv_list(0.03)
dr_05 = get_npv_list(0.05)
dr_07 = get_npv_list(0.07)

plt.plot(x_list, dr_02, label='2%')
plt.plot(x_list, dr_03, label='3%')
plt.plot(x_list, dr_05, label='5%')
plt.plot(x_list, dr_07, label='7%')
plt.title('NPV curves')
plt.xlabel('(years)')
plt.ylabel('(yuan)')
plt.legend()
plt.show()

Note: 7% 折现率也是有说法的。它是 OMB 提供的参考值，维基百科 原文如下：

“The US government uses a variety of discount rates but something around 7% is what the US Office of Management and Budget (OMB) recommends for a pretax rate of return on private investments. In the United Kingdom, HM Treasury fixes the social discount rate for the public sector at 3.5%.”

四、内部收益率IRR

还有一个指标，也可以作为购买保险的参考依据。它叫 IRR，IRR 的数值越高，一般对顾客越有利。

内部收益率 (IRR) 是净现值 (NPV) 等于 0 时的折现率。该定义结合 NPV 公式可得如下代码：

import numpy as np
from scipy.optimize import fsolve

def calc_root(t=50):
    f = lambda e: 500000/((1+e)**t) - sum([9000/((1+e)**i) for i in range(30)])
    return fsolve(f, 0)[0]

root = calc_root()
print(root)

运行上述代码，输出：$root \approx 0.017$

这意味着，80 岁时身故（对应参数 t=50）对应的内部收益率 IRR 约为 0.017。PS: 此数值仅供参考，因为它没有考虑保险兑付，只考虑身故返利。

五、简单结论

一通计算下，这款产品不像看上去那么赚，但也不至于割韭菜。毕竟在重疾险之外，它还提供了返款的机会，虽然不知道真返起来痛不痛快。

险保不炒，保险要回归保险本质。还是要考察它作为重疾险的品质和那 50 万具体的返款的条件才知道值不值。这涉及到一大堆复杂的概率分布计算，本文是不打算解决这个问题的，本文只想给一个简单但稍微反直觉的结论，给看客们一个可供参考的依据：如果你活到 80 岁，期间保险公司兑付 6 万保险，并且最后成功返款，那这就是标准的营亏平衡点。高于这个点你赚了，低于这个点保险公司赚了，你俩是零和游戏。

六、后日谈

“看起来赚，返款时还不知怎样”是这类产品的特点。身为一个风险厌恶者，我更喜欢干净的保险产品：不要返款，也不要理财属性。因为这终究是噱头，在层层的条款背后，大概率的结果是：你玩不过它。

毕竟它是大机构，你们之间产生纠纷，身为个体只会输多赢少。所以我们的策略应该是减少一切和大机构发生纠纷的可能。保险合同，简单最美。
