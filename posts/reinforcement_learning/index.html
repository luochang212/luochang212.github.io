一、Policy Gradient (Review)

1）Proximal Policy Optimization (PPO)

  1. Policy Gradient
  2. On-policy => Off-policy
  3. Add constraint

2）Basic Components

  • Actor: 角色，采取什么行动？
  • Env: 环境，当前状态下提供给Actor什么信息？
  • Reward Function: 给当前行为什么奖励？这个是难点

3）Policy of Actor

Policy is a network

  • input: 游戏画面
  • output: 每个行为对应输出层中的一个神经元

一场游戏叫做一个 episode

4）Expected Reward

每个 episode 出现的概率：

$p\left(s_1\right) = \prod_{t=1}^T p_\theta\left(a_t \mid s_t\right) p\left(s_{t+1} \mid s_t, a_t\right)$

每个 episode 的奖励 R = episode 所有 action 的 reward 之和。公式表达为：

$R(\tau) = \sum_{t=1}^{T} r_t$

因为给定 Action 和 Env 时，给出的下一个 Env 有随机性，所以 R 是一个随机变量，无法计算 R 的真实值，但可以计算 R 的期望。

我们约定 $\tau$ 代表一个 episode 的 action 组合，那么 Reward 的期望可写成：

$Expected \space Reward = \sum_\tau R(\tau) * P_\theta(\tau)$

5）Policy Gradient

由于 $\nabla f(x) = f(x) \nabla log {f(x)}$

有 $\nabla \bar{R}_\theta \approx \frac{1}{N} \sum_{n=1}^N \sum_{t=1}^{T_n} R\left(\tau^n\right) \nabla \log p_\theta\left(a_t^n \mid s_t^n\right)$

6）Implementation

  1. 收集互动数据，基于数据计算 Gradient
  2. 设置学习率，每次根据 (学习率 * Gradient) 调整参数
  3. update model后再收集数据，再update model，循环多次

7）Tips

  1. 设定 baseline 让 Reward 有正有负，好处是让没随机到的action有更多被抽样到的机会
  2. 一个action会影响此后数个state，且影响逐渐衰减

二、From on-policy to off-policy

1）On-policy -> Off-policy

  • On-policy: The agent learned and the agent interaction with the environment is the same.
  • Off-policy: The agent learned and the agent interacting with the environment is different.

Off-policy 的好处：可以重复使用数据，不用反复与环境交互collect data

2）Issue of Importance Sampling

Importance sampling:

$E_{x \sim p}[f(x)] = E_{x \sim q}[f(x) \frac{p(x)}{q(x)}]$

distribution 不能差距过大

$Var_{x \sim q} \left[f(x) \frac{p(x)}{q(x)}\right] = E_{x \sim p}\left[\left(f(x) \frac{p(x)}{q(x)}\right)^2\right] - \left(E_{x \sim p}\left[f(x) \frac{p(x)}{q(x)}\right]\right)^2$

3）Off-policy Gradient

$\nabla \bar{R}_\theta=E_{\tau \sim p_{\theta^{\prime}}}\left[\frac{p_\theta(\tau)}{p_{\theta^{\prime}}(\tau)} R(\tau) \nabla \log p_\theta(\tau)\right]$
