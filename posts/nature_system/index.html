本篇介绍如何用 Agent-based model 的方法，对一个包含狼、麋鹿、草三个物种的生态系统建模。

GitHub项目链接：nature-system

Agent在这里指个体，Agent-base model意即基于个体的模型。在Agent-based model中，现实里的个体与模型里的虚拟实体(virtual entity)是一一对应的，而模型内个体的状态则根据事先写好的规则迭代。因此这种模型适合对个体的详细情况建模。

需要注意的是，在Agent-based model的框架下建模，并不是变量越多越好，过多的变量会造成调试上的困难。选取的变量，足够支撑起我们研究的问题就可以了。

Note: 除了Agent-base model之外，还有一种常用建模方法叫Equation-based model, 即基于方程的建模方法。这种方法就既不适用于表达每个个体发生的情况，也不适用于表达个体的速度、位置等属性信息。而更适合表达平均发生了什么，以及系统平均到达了一种怎样的状态。举例来说，人口模型就很适合用Equation-based model的方法建模。

模型假设

（一）各物种的属性

模型内有三个物种：狼(wolf)、麋鹿(moose)、草(grass)。此外，狼分为头狼和普通狼。下面这张表说明了头狼、普通狼、麋鹿的属性。

普通狼 √ √ √ √ √
麋鹿 √ √ √ √ √

从上表中可以看出，年龄、食物、位置、速度是头狼、普通狼和麋鹿的共有属性。last_breed指和上一次生育的时间间隔，因为在模型内一个时间间隔表示为一次迭代，因此last_breed是整型变量。此外，头狼特有population属性，用来表示头狼所在狼群的狼的数量；特有range属性，表示狼群活动的范围；特有eaten属性，表示狼群在一次迭代之后吃到的麋鹿的数量。普通狼特有packNo属性，是其所属狼群的头狼在列表中的索引号。普通狼没有last_breed属性，是因为其生育被记入头狼名下，也就是记录在整个狼群名下。

草没有那么复杂，为了模型简便，本模型中的草是不会生长的，吃完就没了。

（二）各物种的行为

头狼、普通狼和麋鹿都具有四种行为，这些行为是：eat, migrate, breed, die。这四种行为在不同的生物种类上的实现是不同的。比如麋鹿的迁徙规则是吃掉所有草，直到自己所处的位置没草吃了，才开始迁徙。而头狼则拥有复杂的迁徙规则，会根据狼群内的种群数量，判断应该按照哪套迁徙规则进行移动。

1.先说死亡(die)。

头狼、普通狼和麋鹿有两种状态：生和死。对于狼有两种死法：饿死或者老死。对麋鹿有三种死法：饿死、老死或者被吃。

关于老死的设定，每一种生物都有固定的年龄上限。比如在本模型中，所有狼的最大年龄是25。因此如果没有中途饿死，所有狼都将在第25次迭代时老死。至于饿死，是在动物的food属性为0时发生。

对于狼，死亡的情况还会更复杂一点。狼群的食物是以整个狼群来计算的，当整个狼群的食物低于生存下限时，会有一只狼死亡。

当头狼死去时，如果普通狼的数量不为0，会有一只普通狼晋升为头狼。

2.再说生育(breed)。

不同的物种有不同的生育规则。

每只麋鹿一次只生一个。生育会将母亲的food属性值分一半给孩子。生育有时间间隔，不能持续生育。麋鹿在一次生育后，起码要经历10次迭代，才能生育第二次。生育还有食物限制，当food属性高于10，麋鹿才会生育。如果同时满足时间间隔和食物条件，那么麋鹿一定会进行生育。

本模型将整个狼群看作一个生育单位。当食物有富余时，狼群才会生育。生育个数与狼群群内个体的个数成比例。在不同的群内个体数量水平下，比例系数也不同。生育后，母体的食物也会按比例减少，并将这部分食物赋予孩子。和麋鹿一样，狼群的生育也有时间间隔的限制。一次生育后，起码要经历5次迭代的时间，才能进行下一次生育。

3.然后是迁徙(migrate)。

狼群的迁徙规则取决于头狼，头狼相当于狼群系统的神经中枢。它接收了上一轮狼群的捕猎信息，并安排下一步狼群的走向。但是，头狼无法获知整张地图的麋鹿分布信息，它只能从已知信息推断下一步应该向哪里移动。

具体来讲，狼群的迁徙规则可分为两类讨论。如果在上一次迭代中，整个狼群没有吃到一只麋鹿，那么狼群将随机移动，以寻找麋鹿。如果起码吃到了一只，那么就用一个for循环，找出是狼群中的哪只狼吃到了麋鹿。在下一步迭代中，头狼将移动到那只狼的位置。

头狼移动之后，头狼手下的普通狼将在头狼的range属性值代表的距离范围内四散分布。因为这里引入了两个随机变量，因此普通狼相对于头狼的位置是随机的。值得一提的是range值取决于狼群群内个体数量，数量大的狼群range值也较大。

麋鹿当且仅当在草吃完以后移动。草吃完以后，如果在搜索范围内有草，则向该方向收敛。搜索范围内没草，随机搜索。

PS: 本模型有地理范围限制，所有生物均不可以到达地图之外的地方。因此所有物种在实际迁徙之前，都必须计算移动的终点是否超越了地图边界。

4.最后是捕猎(eat)。

无论是狼还是麋鹿，食物水平都会随着时间的流逝而下降。

对于麋鹿，每次迭代，吃不到食物时，它的food属性值都会减1。而吃到食物时，food属性值会加2，但减去随时间流逝下降的1，实际上food值只增加了1。

对于狼，它的速度值同时也是它的搜索范围。在搜索范围内的麋鹿和狼的距离越近，越容易被判被狼捕获。具体实现如下：

        pk = 1 - (d/spd)
        if pk > rand
            {麋鹿死，狼食物加1，狼移动到麋鹿原先的位置}
        end

这里，pk是被杀的概率的意思。d是麋鹿和狼之间的距离，spd是狼的速度，rand是一个介于0到1之间的随机数。距离短，狼的速度大，pk就大，事件pk>rand发生的可能性就大，因此麋鹿被捕获的概率高。

吃到麋鹿时，food值加2。同时因为本次迭代掉的那1点food值，实际只增加1点。

整体架构

代码有34个m文件，总计1846行，因此无法详述，只能说明大致的框架。

下面用流程图表示了文件之间的调用关系。其中ecolab.m是主文件。左边一列以create打头的，是创建模型的参数和数据结构。agent_solve通过调用各生物的eat, breed, migrate, die文件，刻画了一次迭代中各生物的动作和状态。agent_solve最后还调用了update_messages，以更新各生物在全局变量MESSAGES中的值。然后调用plot_reasult，绘制出各生物的数量变化状态，以及位置变化情况。

本质上，该模型维护着几个比较重要的变量和全局变量，比如agent和MESSAGES。确保这几个变量不出错，即使模型假设有错，整个模型依旧能够正常运行。

代码实现

1.下面这段代码在每次迭代之前，都要执行。这里的“迭代”指的是模型对一次时间间隔的模拟。

function [agent] = initial_iteration(agent,nf)
% This a new function to initial the property at the beginning of each iteration

global MESSAGES PARAM

%% initial alphaWolf.eaten at the beginning of each iteration
%eaten is the number of moose be eaten at each iteration, this
%number will be useful to decision making process of migrate.
for p = 1:length(agent)
    if isa(agent{p},'alphaWolf')
        agent{p}.eaten = 0;
    end
end

%% initial wolf.migration at the beginning of each iteration
% wolf.migration
for q = 1:length(agent)
    if isa(agent{q},'wolf')
        agent{q}.migration = 0;
    end
end

%% update the alphaWolf.population of wolf pack for each iteration
for m = 1:length(agent)
    currAgent = agent{m};
    if isa(currAgent,'alphaWolf')
        count = 0;
        for n = 1:length(agent)
            if isa(agent{n},'wolf') & agent{n}.packNo == m & MESSAGES.atype(n) ~= 0
                count = count + 1;
            end
        end
        currAgent.population = count;
        agent{m} = currAgent;
    end
end

%% updatet alphaWolf.range at the beginning of each iteration
for m = 1:length(agent)
    currAgent = agent{m};
    if isa(currAgent,'alphaWolf')
        if currAgent.population <= PARAM.F_SIZE
            nrange = ceil(PARAM.F_SPD/2 * (1 + nf/5));
        else
            nrange = PARAM.F_SPD;
        end
        currAgent.range = nrange;
        agent{m} = currAgent;
    end
end

从这段代码中可以看到一些整个模型都在反复使用的语句。比如：

if isa(agent{p},'alphaWolf')

MATLAB帮助文档是这么解释的：tf=isa(obj, ClassName) returns true if obj is an instance of the class specified by ClassName, and false otherwise. isa also returns true if obj is an instance of a class that is derived from ClassName.

因此，isa判断的是一个变量是否是某个类的一个实例，或者是否为该类的一个子类的实例。

还有一个常用的判断是：

if agent{n}.packNo == m

鉴于packNo表示一只狼所属狼群的头狼的索引值，而m正是某只头狼的索引。这个语句就判断了一只狼是否是索引为m的头狼所在狼群中的狼。

2.下面这段代码出自从agent_solve中

for cn=n:-1:1
    curr=agent{cn};
    if isa(curr,'moose')|isa(curr,'wolf')|isa(curr,'alphaWolf')
        %% eat rules
        % moose
        if isa(curr,'moose')
            [curr,moose_eaten]=eat(curr,cn);
        end
        % wolf
        if isa(curr,'wolf')        %alphaWolf can not chase prey
            [curr,agent]=eat(curr,cn,agent);  %eating rules (mooses eat food, wolves eat mooses)
        end
        if isa(curr,'alphaWolf')
            eaten=curr.eaten;
        end
        
        %% migrate rule
        % if population is large, wolf pack will be active
        flag = 0;
        if isa(curr,'alphaWolf') & curr.population > PARAM.F_SIZE & eaten < ceil(curr.population * 1/2)
            [curr,agent_migrate]=migrate(curr,cn,agent,eaten);              % if no food was eaten, then migrate in search of some
            agent = agent_migrate;                      %up date cell array with modified agent data structure
            flag =1;
            % if population is small, wolf pack will not move so frequently
        elseif isa(curr,'alphaWolf') & curr.population <= PARAM.F_SIZE & eaten < ceil(curr.population * 1/4)
            [curr,agent_migrate]=migrate(curr,cn,agent,eaten);              % if no food was eaten, then migrate in search of some
            agent = agent_migrate;                      %up date cell array with modified agent data structure
            flag =1;
        elseif isa(curr,'moose') & moose_eaten==0
            curr=migrate(curr,cn);
        end
        % if the alphawolf did not migrate,flag is 0
        % in this case,wolf should move randomly to find the food
        if isa(curr,'alphaWolf') & flag == 0
            for p = 1:length(agent)
                if isa(agent{p},'wolf') & agent{p}.packNo == cn & MESSAGES.atype(p) ~= 0 & agent{p}.migration == 0
                    [agent{p}] = migrate(agent{p},p,agent);
                end
            end
        end
        
        %% death rule
        % wolf (from old age)
        % moose (from starvation or old age)
        if isa(curr,'moose')|isa(curr,'wolf')
            [curr,klld]=die(curr,cn);
        end
        
        % alphaWolf (from old age) and wolf pack (from starvation)
        % PS: alphaWolf represent for the whole wolf pack here
        if isa(curr,'alphaWolf')
            [curr,klld,agent]=die(curr,cn,agent);
        end
        
        %% breed rule
        % moose
        if isa(curr,'moose')&klld==0
            new=[];
            [curr,new]=breed(curr,cn);          %breeding rule
            if ~isempty(new)                    %if current agent has bred during this iteration
                n_new=n_new+1;                 %increase new agent number
                agent{n+n_new}=new;         %add new to end of agent list
            end
        end
        
        % alphaWolf
        % only alphaWolf can breed, wolf can not
        if isa(curr,'alphaWolf')&klld==0
            new=[];
            [curr,new]=breed(curr,cn);          %breeding rule
            if ~isempty(new)                    %if current agent has bred during this iteration
                for ite = 1:length(new)
                    n_new=n_new+1;                 %increase new agent number
                    agent{n+n_new}=new{ite};            %add new to end of agent list
                end
            end
        end
        agent{cn}=curr;                          %up date cell array with modified agent data structure
    end
end

因为之后的计算可能需要用到之前计算的结果，因此各个部分计算的先后次序不宜变动，本模型是按照eat, migrate, die, breed的顺序进行计算。

值得注意的是，第一句的for循环，也因为agent数组的缘故被写成了倒序形式：

for cn=n:-1:1

我的贡献是基于老师的模型，为狼群写了一个集群行为。在@文件内，对每个生物的属性和行为的具体调整，在agent_solve里都可见端倪。可以以此为索引，看看每个生物的行为模式。
