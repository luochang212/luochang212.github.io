一、文件结构介绍

打包前，需要构建以下文件结构：

packaging_tutorial/
├── LICENSE
├── pyproject.toml
├── README.md
├── src/
│   └── YOUR_PACKAGE_HERE/
│       ├── __init__.py
│       └── example.py
└── tests/

文件夹 / 文件 说明：

  • packaging_tutorial: 项目文件夹
  • LICENSE: 开源协议，可由 GitHub 生成
  • pyproject.toml: 打包配置文件
  • README.md: 说明文档
  • src: 项目结构要求有此文件夹
  • YOUR_PACKAGE_HERE: 包的主程序，以“你的包名”命名文件夹

二、添加配置文件

将下面的内容修改一下，改成你的名字邮箱什么的，存成 pyproject.toml 文件

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "example_package_YOUR_USERNAME_HERE"
version = "0.0.1"
authors = [
  { name="Example Author", email="author@example.com" },
]
description = "A small example package"
readme = "README.md"
requires-python = ">=3.7"
classifiers = [
    "Programming Language :: Python :: 3",
    "License :: OSI Approved :: MIT License",
    "Operating System :: OS Independent",
]

[project.urls]
"Homepage" = "https://github.com/pypa/sampleproject"
"Bug Tracker" = "https://github.com/pypa/sampleproject/issues"

三、打包

3.1 准备打包环境

python3 -m pip install --upgrade pip
python3 -m pip install --upgrade build

3.2 打包

在 pyproject.toml 文件所属目录下，打开命令行，运行代码：

python3 -m build

此操作在当前目录下生成名为 dist 的文件夹，里面是打包好的项目。

四、上传

4.1 准备上传环境

python3 -m pip install --upgrade twine

4.2 注册 PyPI 账号

PyPI 提供了测试服和正式服。测试上传选测试服，正式上传选正式服。

账号注册链接 获取 API token 的链接
测试服 test.pypi.org/account/register test.pypi.org/manage/account
正式服 pypi.org/account/register pypi.org/manage/account

注册账号后，获取 API token.

Note: 正式服需要先完成 2FA 验证才能获取 API token

4.3 上传到 PyPI

分两种情况：

1）上传测试服 TestPyPI：

python3 -m twine upload --repository testpypi dist/*

2）上传正式服 PyPI：

twine upload dist/*

运行以上代码，命令行会要求你输入 username 和 password。

  • username 填 __token__
  • password 填你的 API token，就是 pypi- 开头那串东西

4.4 测试

现在可以用 pip install xxx 下载并测试了！
